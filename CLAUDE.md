# Nagi — 凪

撮影旅行のための個人用プランニングPWA。日没・月・天の川・雲量・潮汐を1画面で見て、
星撮り・干潟・ゴールデンアワーのタイミングを判断する。オーナーは写真/動画/旅行の
コンテンツクリエイター(Sony α7C II + Osmo Action 5 Pro、フィルム/アーストーン志向)。

## 言語・コミュニケーション
- 応答・コードコメント・コミットメッセージ・ドキュメントはすべて日本語で書くこと
- 実装前に方針を短く提案し、合意してから書く(このプロジェクトの流儀)

## 設計思想(変更しないこと)

1. **単一HTML完結・オフライン第一**: 天文計算は外部依存ゼロで動く。電波のない山で開くツール。
   ライブラリはCDN参照せず `index.html` に埋め込む(SunCalc v1.9.0 埋め込み済み)。
2. **予報は「あれば嬉しい」扱い**: 雲量・潮位が取得できなくても天文データだけでUIが成立する。
   取得失敗は静かにスキップし、バナーで一度だけ知らせる。
3. **ホワイト×シンプルモダン**: 白背景、余白と罫線、夜のタイムラインバーだけが濃い。
   デザイントークンは `:root` のCSS変数に集約(--amber #B07E3E が唯一のアクセント)。
4. **時間窓は「正午→翌正午」**: 夜を分断しないため。全機能この窓に揃える。

## アーキテクチャ

```
index.html            アプリ本体(CSS/JS込み、SunCalc埋め込み)
sw.js                 Service Worker: 本体=キャッシュ優先 / open-meteo=ネット優先・圏外は最終取得値
manifest.webmanifest  PWAマニフェスト
icons/                水平線×月の潮アイコン(Nagi.png)
```

### index.html 内の主要関数
- `sampleNight(spot, baseNoon, stepMin)` — 正午→翌正午を等間隔サンプリング(太陽高度・月高度・銀河中心高度/方位)
- `gcPos(date, lat, lng)` — 銀河中心(RA 266.417°, Dec -29.008°)の高度方位。恒星時から自前計算
- `nightScore(s)` — 星空適性 0-100。「太陽<-18° かつ 月なし(または月明かり減点 1-frac*1.25)」の
  時間を積分、3時間で100。**天の川の高度はスコアに入れない**(別枠表示)。
  雲量取得時は `eff = score × (1 - cloud/100)` を実効スコアとして表示
- `fetchWeather` / `nightWx` — Open-Meteo forecast API。夜間20:00-04:00の平均雲量・最大降水確率
- `fetchTide` / `tideWindow` / `tideExtremes` — Open-Meteo Marine API (sea_level_height_msl)。
  陸グリッドに当たったら周囲±0.05°の海を自動探索。極値は隣接3点の放物線補間(検証誤差±1分)
- `drawTimeline` / `drawTide` — canvas描画。**潮位カーブはタイムラインと同じx軸**(時刻整合が肝)
- `compassSVG(mwAz, ssAz, srAz)` — 天の川方位(針)+日没/日出方位(点)
- `store` — ストレージ抽象化: claude.aiアーティファクト内は window.storage、
  ホスティング時は localStorage に自動フォールバック。キーは `spots-v1`

## 運用ルール

- **sw.js の `CACHE` バージョンをリリースごとに上げる**(上げ忘れると更新が配信されない)
- 潮位は海洋モデル予測。UIに「気象庁潮位表(那智勝浦=浦神)でダブルチェック」の注意書きがあり、
  これは消さない
- 座標・文言は日本語UI、時刻はデバイスのローカルタイム(実質JST)

### プロダクトの背骨(v6で確定・以降ぶれさせない)
主役は個別機能ではなく「計画→撮影→振り返り」の連続性。その芯が「ショット」。見やすさ(共有相手には
旅のしおり)とニッチさ(自分にはショット+光+潮の深い層)は、対立させずレイヤーの深さで両立する。
