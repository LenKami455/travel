# v7 設計書 — ロケーション資産化

v6で積んだ `shots[]` を、スポット単位の撮影履歴として「見せる」だけの機能。新しいデータ構造は増やさない。

## スコープ

スポット詳細画面(旅程に依存しない `renderCards()` のカード)に「📷 撮影履歴」の折りたたみセクションを追加する。
そのスポットで `status: 'shot'`(撮影済み)になっているショットを、旅を問わず日付の新しい順に一覧表示する。

各行の表示: 撮影日・タイトル・その夜の条件(月齢輝面%・星空スコアを `shot.date` から再計算)・
`captured.time` / `captured.files` / `captured.link`(あれば作例リンク)。カード見出しにも
「📷 N」の件数バッジを出す。

非スコープ: 潮位の履歴表示(Open-Meteoは予報のみでオフライン再計算ができないため対象外)、
新規データ項目の追加(v6の `captured` に条件スナップショットを足す案は採らず、都度再計算する)。

## 実装方針

条件(月齢・スコア)は保存せず、`shot.date` から `sampleNight` / `nightScore` を毎回re実行して求める。
天文計算は決定論的でオフライン完結なので、過去日でも保存済みデータなしに再現できる。これにより
v6が「任意」としていた `captured` へのスナップショット追記を省略できた。

- `shotHistoryFor(sp)`: `sp.shots` から `status==='shot'` を抽出し、`date` があれば条件を再計算して
  付加、日付降順でソート(データ層)
- `renderHistorySection(history)`: 履歴HTMLを組み立て(空なら「まだありません」を表示)
- `renderCards()` に上記2つを組み込み、カード見出しに件数バッジを追加

## テスト観点

`shots` 未定義スポット(v6以前のデータ)でも例外なく描画されること。`planned`/`skipped` が履歴に出ないこと。
日付なしショットは条件なしで最後尾に出ること。タイトル・リンクのエスケープ(XSS対策)。

## v8への布石

旅程に「タグ」(星撮り/干潟/ゴールデンアワー等)を持たせ、タグから機材チェックリストを提案するのがv8。
今回はタグの仕組み自体は作っていない。
