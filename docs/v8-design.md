# v8 設計書 — 機材プランナー

旅程に「タグ」を持たせ、タグから持ち物チェックリストを生成する。v5〜v7とは独立した新しいデータ層。

## スコープ

旅程(`trip`)にタグ(星撮り/干潟/ゴールデンアワー)と持ち物チェックリストを追加する。タグはプリセットの
持ち物セットを持ち、「タグから更新」で不足分だけを追加(マージ)する。チェック状態・自由追加項目は
タグの選択が変わっても消えない。

非スコープ: タグ・プリセット自体の編集UI(コード内定数として固定)、旅程作成フォームでのタグ初期選択
(作成後にトグルで選ぶ想定)。

## データ設計

```jsonc
// trips-v1 の各旅程に追加
{
  "id": "tr_x9y8", "name": "...", "start": "...", "days": [...],
  "tags": ["astro", "tidal"],              // 選択中のタグ(複数可)
  "gearChecklist": [                        // 実体リスト。タグを外しても手動で消すまで残る
    { "id": "gear_k3m9", "text": "三脚", "checked": false, "tag": "astro" },
    { "id": "gear_p0q1", "text": "予備SDカード", "checked": true, "tag": null }  // tag:null = 自由追加
  ]
}
```

タグ定義は `GEAR_TAGS` 定数(コード内、`index.html`):

```js
const GEAR_TAGS = [
  { id:"astro",  label:"🌌 星撮り",       items:[...] },
  { id:"tidal",  label:"🌊 干潟",         items:[...] },
  { id:"golden", label:"🌅 ゴールデンアワー", items:[...] },
];
```

マイグレーション: 起動時、`tags`/`gearChecklist` を持たない旅程に `[]` を付与(冪等)。v5/v6と同じ場所
(`loadTrips`)に付け足すだけ。

## UI

旅程ビュー上部(撮影チェック/振り返りと並び)に「🎒 持ち物」トグルを追加。ONにすると:

1. タグのチェックボックス群(複数選択、選択即保存)
2. 「タグから更新」ボタン — 選択中タグの持ち物のうち、まだリストに無いテキストだけを追加(既存の
   チェック状態・自由追加項目は保持。テキスト一致で重複排除)
3. チェックリスト本体(チェックON/OFF、削除、由来タグのラベル表示)
4. 自由入力欄(Enterまたはボタンで追加)

fieldMode/reviewModeと同様の排他トグル(3つのうち1つだけON)。

## レイヤー分離(v6の方針を継承)

共有リンク(`shareTrip`)では `tags` / `gearChecklist` を旅程データから除外する。持ち物は個人の機材準備層
であり、旅のしおりを見る同行者に見せる情報ではないため。取り込み側(`importSharedTrip`)は空の
`tags:[]` / `gearChecklist:[]` で開始する。

## テスト観点

タグ未定義の旧データ(v7以前)でも例外なく描画されること。「タグから更新」の重複排除(同じタグで
2回押しても増えない)。タグを外してもチェックリスト本体は消えないこと。自由追加・チェック・削除が
タグ由来アイテムと同様に動くこと。持ち物モードのUIを再描画する経路(タグ切替・更新ボタン)が
`entryList` を毎回クリアしてから描き直すこと(appendの重複防止)。
